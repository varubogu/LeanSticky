# LeanSticky 設計書

## 1. 目的
- 軽量な付箋デスクトップアプリを作る。
- メモリ使用量の小ささ、起動の速さ、単純なローカル保存を最優先にする。
- Rust の共通コードベースで Windows、macOS、Linux をサポートする。
- GUI アプリと TUI アプリの両方を同じ保存モデルで提供する。
- 初期リリースから日本語と英語に対応する。

## 2. 対象範囲
### デスクトップ GUI
- Rust + egui/eframe で実装する。
- OS 標準のウインドウ枠は使わず、独自のフレームレス UI を使う。
- 常に手前表示の切り替え、最小化、最大化/復元、閉じるをサポートする。
- 操作ボタンは通常は隠し、ホバー時に浮き上がるアニメーションで表示する。
- ウインドウは自由に移動・リサイズできる。
- テキストは入力停止 1.5 秒後に自動保存する。
- 自動保存ごとに履歴スナップショットを 1 件追加する。
- タスクトレイ常駐に対応し、クリックで付箋一覧、右クリックでメニューを表示する。
- 右クリックメニューには最低限 `Settings` と `Quit` を用意する。
- 画面に表示される文字列は多言語メッセージから読み込む。

### TUI
- tmux などの端末環境で動作する。
- GUI 版と同じノートファイル、履歴、同期ロジックを共有する。
- GUI と同じメッセージキーを使って表示言語を切り替える。
- グラフィカルなウインドウ操作ではなく、閲覧・編集の速さを重視する。

## 3. 非機能要件
- 軽量性を最優先とし、依存を増やしすぎない。
- オフラインファースト、ファイルベースで動作する。
- 設定ファイルと付箋データの両方に YAML を使う。
- 設定ファイルはローカルにのみ保持し、同期対象のノートフォルダとは分離する。
- Dropbox、iCloud Drive、OneDrive、Syncthing などの同期フォルダでも扱いやすくする。
- 外部でファイルが変更されたら自動で再読み込みする。
- 部分書き込みや同期競合が起きても、可能な限り安全に復旧できるようにする。

## 4. アーキテクチャ
- `core`: ドメインロジック、YAML 永続化、履歴、ファイル監視、設定、多言語読み込み
- `desktop`: egui ベースの GUI、独自ウインドウ操作、トレイ連携
- `tui`: 同じ `core` を使う端末 UI

`core` は GUI フレームワークに依存させない。

## 5. 多言語対応
### メッセージソース
UI 文字列は `messages.yml` に定義する。

形式:
```yaml
sample_key:
  ja: サンプルメッセージ
  en: sample messages
```

### ルール
- 各メッセージキーは `ja` と `en` を持つ。
- v1 は日本語と英語のみを必須とする。
- 翻訳が欠けた場合は英語へフォールバックする。
- 表示言語はローカル設定から選び、必要ならシステムロケールを初期値に使う。
- `messages.yml` は公開スキーマで検証できるようにする。

## 6. 保存モデル
### ローカル設定
設定ファイルはプラットフォームごとのローカル設定ディレクトリに置き、ノートデータとは同期しない。

例:
- Windows: `%AppData%/LeanSticky/config.yml`
- macOS: `~/Library/Application Support/LeanSticky/config.yml`
- Linux: `${XDG_CONFIG_HOME:-~/.config}/leansticky/config.yml`

推奨フィールド:
- `schema_version`
- `locale`
- `note_roots`: 読み込むフォルダパスの配列
- `autosave_delay_ms`
- `history`

### ノートルート
ローカル設定には任意の数のノートルートフォルダを設定できる。

各ノートルートでは YAML ファイルを使う:
```text
<note-root>/
  notes/
    <note-id>.yml
  history/
    <note-id>/
      <timestamp>.yml
```

### ノートファイル
各ノートは UTF-8 の YAML ファイル 1 つで保存する。

推奨フィールド:
- `schema_version`
- `id`
- `title`
- `content`
- `created_at`
- `updated_at`
- `always_on_top`
- `window`: 位置、サイズ、最大化状態

### この方式を選ぶ理由
- マシン固有設定をローカル限定にできる。
- 複数フォルダ対応で用途ごとにノート保存先を分けられる。
- YAML は目視しやすく、手作業でも扱いやすい。
- 履歴は追記型にしやすく、ポリシーで間引ける。

## 7. 保存と履歴のルール
- 編集のたびに 1.5 秒のアイドルタイマーを開始する。
- 追加入力があればタイマーをリセットする。
- タイマー満了時に以下を行う。
  - 現在のノートを YAML としてアトミックに保存する
  - YAML の履歴スナップショットを 1 件追加する
  - メモリ上の dirty 状態を更新する
- 可能な環境では一時ファイル + rename によるアトミック保存を使う。
- 履歴は無制限に増やさず、例えば以下のような上限を設ける。
  - 直近 1 時間は高密度
  - 古い履歴は間引く
  - ノートごとに最大件数を持つ

## 8. 同期と再読み込み
- 設定されたすべてのノートルートを監視する。
- 外部変更を検知した場合:
  - ローカル未保存がなければ自動再読み込みする
  - ローカル編集中ならメモリ上の内容を保持し、競合状態を表示する
- 外部削除を検知した場合:
  - 設定された確認ルールに従って一覧から外す
- 新規ファイル追加を検知した場合:
  - 自動でノート一覧に読み込む
- 実用上問題なければローカル設定ファイルの変更も再起動なしで再読み込みする

v1 の競合対応は複雑にしすぎない。
- 破壊的な上書きを避ける
- ローカル未保存テキストを保持する
- `Reload from disk` と `Save as new version` を提供する

## 9. デスクトップ UI
### 基本方針
- 視覚的な重さを小さくする。
- ノートは文書アプリではなく、浮かぶ道具のように見せる。
- 独自 UI でもキーボード操作性を失わない。

### ウインドウ操作
- 常に手前表示を切り替えるピンアイコン
- 最小化
- 最大化/復元
- 閉じる
- 各ボタンは通常時に隠れ、ホバー時に軽い上昇・フェードで現れる

### ノート表示
- メイン領域は低遅延なテキストエディタとする。
- 必要ならコンパクトなタイトル欄を持つ。
- 自動保存なので、dirty 状態の表示は控えめにする。
- ラベル、メニュー、ステータス文言は `messages.yml` から取得する。

## 10. トレイ動作
- 左クリック: 付箋一覧とクイック操作を開く
- 右クリック: 多言語化された `Settings` と `Quit` を含むメニューを開く
- すべての付箋ウインドウを閉じてもトレイ常駐は維持する

## 11. TUI 設計
- GUI と同じノートルートとローカル設定を使う。
- 一覧、開く/編集、検索、作成、削除、履歴閲覧をサポートする。
- マウス前提にしない。
- GUI と同じメッセージキーと表示言語選択を使う。
- tmux などのマルチプレクサ内で扱いやすくする。

## 12. 公開スキーマ
docs は GitHub Pages で公開する前提とする。

スキーマは `docs/schema/` 配下にバージョン付きで置き、以下のようにアクセスできるようにする。
- `/schema/v001/schema.json`
- `/schema/v001/config.schema.json`
- `/schema/v001/note.schema.json`
- `/schema/v001/messages.schema.json`

用途:
- `schema.json`: バージョン入口と共通定義
- `config.schema.json`: ローカル設定 YAML 用
- `note.schema.json`: ノート本体と履歴 YAML 用
- `messages.schema.json`: 多言語メッセージ YAML 用

## 13. 推奨依存
- GUI: `eframe` / `egui`
- 永続化: `serde` + `serde_yaml`
- ファイル監視: `notify`
- トレイ連携: `tray-icon`
- ローカル設定パス: `directories`
- `tokio` は本当に必要になるまで入れない

## 14. マイルストーン
1. `core` クレートで YAML ノートモデル、YAML 設定、履歴、多言語読み込み、アトミック保存を実装
2. 自動保存付きの基本 GUI ノート画面を実装
3. 独自ウインドウ操作と常に手前表示切り替えを実装
4. トレイ連携と多言語ノート一覧を実装
5. 複数ノートルート監視、外部変更の再読み込み、競合処理を実装
6. `core` を使う TUI を実装
