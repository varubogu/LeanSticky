# LeanSticky 設計書

## 1. 目的
- 軽量な付箋デスクトップアプリを作る。
- メモリ使用量の小ささ、起動の速さ、単純なローカル保存を最優先にする。
- Rust の共通コードベースで Windows、macOS、Linux をサポートする。
- GUI アプリと TUI アプリの両方を同じ保存モデルで提供する。
- 初期リリースから日本語と英語に対応する。

## 2. 対象範囲
### デスクトップ GUI
- Rust + egui/eframe で実装する。
- 1 プロセスのみ起動し、管理ウインドウから複数ノートウインドウを扱う。
- 初回実装では OS 標準ウインドウ枠を使い、複数ノート表示、自動保存、終了時復元を優先する。
- 将来的にフレームレス UI、トレイ常駐、常に手前表示などを追加する。
- テキストは入力停止 1.5 秒後に自動保存する。
- 自動保存ごとに履歴スナップショットを 1 件追加する。
- 画面に表示される文字列は多言語メッセージから読み込む。

### TUI
- tmux などの端末環境で動作する。
- GUI 版と同じノートファイル、履歴、同期ロジックを共有する。
- GUI と同じメッセージキーを使って表示言語を切り替える。
- 初回実装では crate の土台のみを用意し、実用 UI は後続マイルストーンで追加する。

## 3. 非機能要件
- 軽量性を最優先とし、依存を増やしすぎない。
- オフラインファースト、ファイルベースで動作する。
- 設定ファイル、セッション状態、付箋データに YAML を使う。
- 設定ファイルとセッション状態はローカルにのみ保持し、同期対象のノートフォルダとは分離する。
- Dropbox、iCloud Drive、OneDrive、Syncthing などの同期フォルダでも扱いやすくする。
- 外部でファイルが変更されたら自動で再読み込みする設計を維持し、実装は後続マイルストーンで追加する。
- 部分書き込みや同期競合が起きても、可能な限り安全に復旧できるようにする。

## 4. アーキテクチャ
- `core`: ドメインロジック、YAML 永続化、履歴、設定、多言語読み込み、起動時 bootstrap
- `desktop`: egui ベースの GUI、管理ウインドウ、複数ノートウインドウ、自動保存、単一起動制御
- `tui`: 同じ `core` を使う端末 UI

`core` は GUI フレームワークに依存させない。

## 5. 多言語対応
### メッセージソース
UI 文字列は `messages.yml` に定義する。

形式:
```yaml
sample_key:
  ja: サンプルメッセージ
  en: sample message
```

### ルール
- 各メッセージキーは `en` を必須とする。
- v1 は `ja` と `en` の 2 言語をサポートする。
- 翻訳が欠けた場合は英語へフォールバックする。
- 表示言語はローカル設定から選び、必要ならシステムロケールを初期値に使う。
- `messages.yml` は公開スキーマで検証できるようにする。
- 初回実装ではアプリ同梱の `messages.yml` を読み込む。

## 6. 保存モデル
### ローカル設定
設定ファイルはプラットフォームごとのローカル設定ディレクトリに置き、ノートデータとは同期しない。

例:
- Windows: `%AppData%/LeanSticky/config.yml`
- macOS: `~/Library/Application Support/LeanSticky/config.yml`
- Linux: `${XDG_CONFIG_HOME:-~/.config}/leansticky/config.yml`

フィールド:
- `schema_version`
- `locale`
- `note_roots`: 読み込むフォルダパスの配列
- `autosave_delay_ms`
- `history.max_snapshots_per_note`

### ローカルセッション
管理ウインドウ位置、開いているノート一覧、各ノートウインドウの復元状態は `session.yml` に保持する。

例:
- Windows: `%AppData%/LeanSticky/session.yml`
- macOS: `~/Library/Application Support/LeanSticky/session.yml`
- Linux: `${XDG_CONFIG_HOME:-~/.config}/leansticky/session.yml`

フィールド:
- `schema_version`
- `management_window`
- `open_windows[]`

各 `open_windows` は以下を持つ:
- `root`
- `note_id`
- `position`
- `size`
- `maximized`

### ノートルート
ローカル設定には任意の数のノートルートフォルダを設定できる。

各ノートルートでは YAML ファイルを使う:
```text
<note-root>/
  notes/
    <note-id>.yml
  history/
    <note-id>/
      <timestamp>.yml
```

### ノートファイル
各ノートは UTF-8 の YAML ファイル 1 つで保存する。

フィールド:
- `schema_version`
- `id`
- `title`
- `content`
- `created_at`
- `updated_at`

ノートファイルには機械依存のウインドウ位置やローカル復元状態を入れない。

### この方式を選ぶ理由
- マシン固有設定とセッション状態をローカル限定にできる。
- ノート本文は同期フォルダで安全に共有しやすい。
- 複数フォルダ対応で用途ごとにノート保存先を分けられる。
- YAML は目視しやすく、手作業でも扱いやすい。
- 履歴は追記型にしやすく、ポリシーで間引ける。

## 7. 保存と履歴のルール
- 編集のたびに `autosave_delay_ms` のアイドルタイマーを開始する。
- 追加入力があればタイマーをリセットする。
- タイマー満了時に以下を行う。
  - 現在のノートを YAML としてアトミックに保存する
  - YAML の履歴スナップショットを 1 件追加する
  - メモリ上の dirty 状態を更新する
- 初回実装では履歴間引きは `max_snapshots_per_note` の件数上限のみ適用する。
- 可能な環境では一時ファイル + rename によるアトミック保存を使う。

## 8. 起動・復元
- 初回起動では platform data dir 配下に既定のノートルートを作る。
- `config.yml` と `session.yml` が無ければ既定値で自動生成する。
- ノートが 0 件なら空ノートを 1 件自動作成する。
- 起動時は `session.yml` の `open_windows` を復元する。
- 復元対象のノートが見つからない場合は除外する。
- 復元対象が 0 件になった場合は既存先頭ノートを開き、さらにノートが無ければ新規作成する。
- `open_windows` の並び順を復元順として扱う。

## 9. デスクトップ UI
### 基本方針
- 視覚的な重さを小さくする。
- ノートは文書アプリではなく、浮かぶ道具のように見せる。
- 管理ウインドウとノートウインドウを分け、複数ノートを扱いやすくする。

### 初回実装
- 管理ウインドウで一覧表示、新規作成、既存ノートを開く操作を提供する。
- 各ノートは同一プロセス内の別ウインドウとして開く。
- 2 回目の起動では既存プロセスへアクティベート要求を送り、新しいプロセスは終了する。
- 既存プロセスは管理ウインドウを前面化する。
- ノートウインドウではタイトルと本文を編集できる。
- 保存状態は控えめなステータス表示にする。

### 後続マイルストーン
- フレームレス UI
- 常に手前表示切り替え
- トレイ常駐
- 外部変更監視

## 10. TUI 設計
- GUI と同じノートルートとローカル設定を使う。
- 一覧、開く/編集、検索、作成、削除、履歴閲覧を最終的にサポートする。
- マウス前提にしない。
- GUI と同じメッセージキーと表示言語選択を使う。
- tmux などのマルチプレクサ内で扱いやすくする。

## 11. 公開スキーマ
docs は GitHub Pages で公開する前提とする。

スキーマは `docs/schema/` 配下にバージョン付きで置き、以下のようにアクセスできるようにする。
- `/schema/v001/schema.json`
- `/schema/v001/config.schema.json`
- `/schema/v001/session.schema.json`
- `/schema/v001/note.schema.json`
- `/schema/v001/messages.schema.json`

用途:
- `schema.json`: バージョン入口と共通定義
- `config.schema.json`: ローカル設定 YAML 用
- `session.schema.json`: ローカル復元状態 YAML 用
- `note.schema.json`: ノート本体と履歴 YAML 用
- `messages.schema.json`: 多言語メッセージ YAML 用

## 12. 推奨依存
- GUI: `eframe` / `egui`
- 永続化: `serde` + `serde_yaml`
- ローカル設定パス: `directories`
- 単一起動: `single-instance`
- ID 採番: `ulid`
- `tokio` は本当に必要になるまで入れない

## 13. マイルストーン
1. `core` クレートで YAML ノートモデル、設定、セッション、履歴、多言語読み込み、アトミック保存を実装
2. 自動保存付きの基本 GUI ノート画面と管理ウインドウを実装
3. 単一起動制御と再起動時のセッション復元を実装
4. 独自ウインドウ操作と常に手前表示切り替えを実装
5. トレイ連携と多言語ノート一覧を実装
6. 複数ノートルート監視、外部変更の再読み込み、競合処理を実装
7. `core` を使う TUI を実装
